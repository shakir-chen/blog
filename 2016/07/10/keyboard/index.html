Ducky mini Keyboard Usage Conclusion

Posted on 2016-07-10   |   In Equipment   |  

As a programmer, there are many reasons to have a good keyboard. Comparing than mouse, input from keyboard has more certainty and consistency, which means you don’t need to find your mouse location and determine which files and buttons to click and you don’t need switch frequently from keyboard to mouse and vice versa. Especially, when your job is not for user’s interface or some art design at computer, keyboard and related softwares count considerably for your creativity and productivity.

One year ago, after becoming a postgraduate at computer simulation direction, I decided to keep away from windows which made me stupid. Mini Ducky Keyboard was bought at that time, which seems a big cost then but now it is a valuable investment for me. Someone may suggest that HHKB(Happy Hacker Keyboard) would be the best for coder. However, HHKB is too expensive, more Unix-friendly and harder to buy. Therefore, for a new hand, mini Duky would be a better choice.

Now, some benificial experence is gained by a hard exploration and I am pleased to share here.

[Ducky Keyboard]http://www.duckychannel.com.tw/en/ducky-mini/

Keyboard Basic Information

Ducky Mini
Mechanical Keyboard, Red Switches, 61 Keys (Professional)
Multiple Background Colors and multi-functional
Produced in Taiwan, Price ~$120

Basic Settings

There are some good mode settings built in the hardware product, which enables
users to personalize a more HHKB-like keyboard.

  • Caps Key -> Ctrl Key
    Fn+Alt+H 3s -> Caps -> choose Ctrl as replacement
    Ctrl key near A is really a better design for coder, providing us to press ctrl
    in a shorter time.

  • `~ key-> Esc Key
    Fn+Alt+H 3s -> -> choose Esc as replacement
    Esc is more frequently used key then `~ key and in vim, a better way to map Esc
    key is to utilize quick type k and j (succeeded by Github k-vim).

Software Adided

Without direction key, there might be a little bit inconvenient at the beginning.
Luckily, even in windows, some easy key mappings could support you in the most
situations.

Text and Doc editor -> Vim, latex
browser -> firefox + vimperator
IDE -> vim + pyclewn + gdb or Emacs
Windows switch and other key mapping -> simple programming with autohotkey
Blogs -> Hexo and edit in vim

Living in vim or shell, I believe, is a comfortable way for programmers. That’s
also a good reason to study Vim instead of IDE tools at the beginning. As a Vimer,
I also recommend Vim rather than Emacs also because of its key design.

Hard to study but it is worth.
